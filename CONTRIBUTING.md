# Contributing

1. Create a branch for the change.
2. Keep the application compatible with Windows PowerShell 5.1 unless the change is documented otherwise.
3. Run `tests\Validate-Project.ps1` before opening a pull request.
4. Do not commit passwords, hashes from real accounts, wordlists containing private data, generated output, or third-party binaries.
5. Describe the behavior changed and the Windows version used for testing.
